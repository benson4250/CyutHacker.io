什麼是 SET？

SET (Social-Engineer Toolkit) 是一個專為社交工程攻擊設計的開源滲透測試框架。它由 TrustedSec 開發，預裝在 Kali Linux 中，是進行安全意識測試和紅隊演練的重要工具。

為什麼要學習 SET？

  • 專業工具 - 專門為社交工程攻擊設計的完整框架
  • 高度自動化 - 簡化複雜的攻擊流程
  • 多向量攻擊 - 支援多種攻擊方式和載荷
  • 業界標準 - 被全球安全專業人士廣泛使用
  • 持續更新 - 擁有活躍的社群支持和持續開發

SET 的歷史與發展

SET 最初由 David Kennedy (ReL1K) 創建，旨在提供一個統一的框架來模擬真實世界的社交工程攻擊。隨著時間的推移，它已經發展成為功能完整的社交工程測試平台，成為滲透測試人員的標準工具之一。

核心概念

在開始使用 SET 之前，了解幾個基本概念會讓學習過程更加順利。

什麼是「社交工程」？

社交工程是利用人類心理弱點而非技術漏洞來獲取敏感資訊或訪問權限的攻擊方法。SET 專門設計用來模擬這些攻擊，幫助組織測試和提升安全意識。

資訊收集
→

→

→
執行攻擊

社交工程攻擊通常遵循特定的心理操縱循環

攻擊分類

  • 魚叉式釣魚：針對特定個人或組織的定制化釣魚攻擊
  • 網站克隆：複製合法網站以竊取憑證
  • 惡意文件：帶有惡意負載的文件，如PDF或Word文檔
  • 多媒體攻擊：利用音頻、視頻等媒體進行攻擊

SET 工作流程

SET 遵循標準化的攻擊流程：目標選擇 → 攻擊向量選擇 → 負載生成 → 交付機制 → 結果收集。

攻擊向量詳解

了解 SET 提供的各種攻擊向量對於有效使用工具至關重要。

魚叉式釣魚

風險等級：嚴重

針對特定目標的定制化釣魚郵件攻擊，成功率最高。

適用場景: 內部安全測試

網站憑證收集

風險等級：高

克隆合法網站並收集用戶輸入的登錄憑證。

示例: Facebook, Gmail 克隆

惡意文件生成

風險等級：高

創建帶有惡意負載的PDF、Word或Excel文件。

QRCode 攻擊

風險等級：中

生成惡意QRCode，引導用戶訪問釣魚網站。

SMS 釣魚

風險等級：中

通過短信發送釣魚連結，針對移動設備用戶。

資訊收集

風險等級：低

收集目標公開資訊，為攻擊做準備。

攻擊成功率統計

根據實際測試數據，不同攻擊向量的成功率：

  • 魚叉式釣魚：23-28% 點擊率
  • 網站憑證收集：15-20% 提交率
  • 惡意文件附件：5-10% 執行率
  • SMS 釣魚：3-7% 回應率

防禦措施與檢測

了解攻擊技術的同時，學習如何防禦和檢測這些攻擊同樣重要。

技術防禦

組織可以實施的技術控制措施：

1

電子郵件安全

實施SPF、DKIM和DMARC協議，檢測和阻止釣魚郵件。

2

Web 過濾

使用Web內容過濾解決方案，阻止對惡意網站的訪問。

3

端點保護

部署先進的端點檢測和響應(EDR)解決方案。

4

多因素認證

實施MFA，即使憑證被竊取也能提供保護。

檢測 SET 攻擊

SET 攻擊的一些特徵指標：

  • 異常的電子郵件發送模式
  • 證書不匹配的網站
  • 可疑的URL結構和重定向
  • 已知的SET用戶代理字串

安全意識培訓

人員是防禦社交工程攻擊的最後一道防線：

  • 定期進行釣魚模擬測試
  • 教育員工識別釣魚跡象
  • 建立明確的事件報告流程
  • 獎勵安全行為

道德與法律規範

使用 SET 和其他滲透測試工具時，必須遵守嚴格的道德和法律規範。

合法授權

在進行任何測試之前，必須獲得明確的書面授權：

  • 高 - 需要立即關注的嚴重漏洞
  • 中 - 重要的安全問題，應該盡快修復
  • 低 - 潛在的安全問題，建議修復
  • 資訊 - 僅供參考的資訊，不一定是安全問題

測試範圍界定

明確界定測試的範圍和限制：

1

目標系統

明確列出允許測試的系統和應用程式。

2

測試時間

規定測試的具體時間範圍。

3

攻擊方法

明確允許使用的攻擊技術和方法。

4

聯絡方式

提供緊急情況下中斷測試的聯絡方式。

負責任披露

發現漏洞後的處理流程：

  • 立即向適當的管理層報告
  • 提供詳細的技術資訊和修復建議
  • 協助驗證修復效果
  • 遵守組織的披露政策

重要法律提醒

請僅在您擁有或已獲得明確授權的系統上使用 SET 進行測試。未經授權的測試可能違反：

  • 電腦詐騙和濫用法案 (CFAA)
  • 歐盟一般資料保護規範 (GDPR)
  • 當地電腦犯罪相關法律

道德黑客守則：

  • 始終獲得適當的書面授權
  • 尊重隱私和資料保護
  • 負責任地披露發現的漏洞
  • 使用技能來改善安全性，而不是破壞它

遵循這些準則，確保您的安全測試活動既有效又合法！
