什麼是 OWASP ZAP？

OWASP ZAP (Zed Attack Proxy) 是一個由 OWASP (開放網路軟體安全計畫) 開發的、全球最受歡迎的開源網站應用程式安全掃描工具。無論您是開發人員、測試人員還是資安初學者，ZAP 都是開始學習網站安全測試的理想選擇。

為什麼要學習 ZAP？

  • 完全免費且開源 - 不用擔心授權費用
  • 功能強大 - 提供從簡單掃描到進階測試的各種功能
  • 易於使用 - 提供友好的圖形化介面，適合初學者
  • 業界標準 - 被全球許多組織和專業人士使用
  • 持續更新 - 擁有活躍的社群支持和持續開發

ZAP 的歷史與發展

OWASP ZAP 項目始於 2010 年，由 Simon Bennetts 創建。最初是作為一個簡單的代理工具，如今已發展成為功能完整的網站安全測試平台。ZAP 現在是 OWASP 的旗艦項目之一，每月有數十萬次下載。

核心概念

在開始使用 ZAP 之前，了解幾個基本概念會讓學習過程更加順利。

什麼是「代理」？

ZAP 是一個「中繼代理」，意思是它位於您的瀏覽器和目標網站之間，可以攔截和檢查兩者之間的所有通訊。

[瀏覽器 <-> ZAP <-> 網站 示意圖]

ZAP 作為瀏覽器與網站之間的中繼代理

被動掃描 vs. 主動掃描

  • 被動掃描：ZAP 監聽瀏覽器與網站之間的通訊，但不主動發送任何請求。這就像「觀察者」模式，安全且不會對網站造成負擔。
  • 主動掃描：ZAP 主動向網站發送專門設計的測試請求，試圖找出安全漏洞。這就像「測試員」模式，更深入但可能對網站造成較大負擔。

工作階段管理

ZAP 能夠管理網站會話，模擬真實用戶的行為。這對於測試需要登入的應用程式至關重要。

環境設定

我們假設您已經有一個準備好的 Kali Linux 環境。讓我們開始設定 ZAP 進行第一次測試。

1

啟動 ZAP

在 Kali Linux 中，您可以通過以下方式啟動 ZAP：

  • 點擊左上角的「Applications」
  • 選擇「03 - Web Application Analysis」
  • 點擊「owasp-zap」
  • 或者，在終端機中輸入：owasp-zap
2

初次設定

第一次啟動時，ZAP 可能會詢問您是否要保存工作階段。選擇「是，我想持續保存這個工作階段」或「不，我不想持續保存這個工作階段」都可以，對初學者影響不大。

3

設定測試目標

為了合法且安全地練習，我們將使用 OWASP 專門為測試設計的網站：

測試網站： https://juice-shop.herokuapp.com

這個網站故意設計了各種安全漏洞，讓我們可以安全地練習掃描技術。

瀏覽器設定

為了讓 ZAP 能夠攔截流量，您需要設定瀏覽器使用 ZAP 作為代理伺服器：

代理伺服器: 127.0.0.1
連接埠: 8080

進行被動掃描

被動掃描是開始安全測試最簡單、最安全的方式。讓我們開始吧！

1

瀏覽網站

確保 ZAP 正在運行，然後打開 Kali 中的 Firefox 瀏覽器，訪問我們的測試網站：

https://juice-shop.herokuapp.com

在網站上隨意點擊幾個連結，嘗試登入（可以使用任意帳號密碼），瀏覽幾個頁面。

2

觀察 ZAP 的變化

回到 ZAP 介面，注意左側的「Sites」視窗。您應該會看到 juice-shop.herokuapp.com 出現，下面有許多節點。這些是 ZAP 自動發現的網站結構。

3

檢查被動掃描結果

點擊下方的「Alerts」標籤頁，您可能會看到一些初步的安全警告。這些是 ZAP 透過被動掃描發現的潛在問題。

被動掃描不會對網站造成負擔，因為它只是分析正常的瀏覽流量。

被動掃描的優點與限制

優點：

  • 不會對目標網站造成負擔
  • 幾乎沒有誤報
  • 可以發現資訊洩漏等問題

限制：

  • 只能發現表面問題
  • 無法檢測深層漏洞
  • 依賴於用戶的瀏覽行為

進行主動掃描

現在讓我們進行更深入的主動掃描，這將幫助我們發現更多潛在的安全漏洞。

1

啟動主動掃描

在 ZAP 左側的「Sites」視窗中，在 juice-shop.herokuapp.com 上點擊右鍵，選擇「Attack」 → 「Active Scan」。

2

監控掃描進度

點擊下方的「Active Scan」標籤頁，您可以看到掃描的進度。ZAP 正在向網站發送數千個測試請求，試圖找出安全漏洞。

這可能需要幾分鐘時間，取決於網站的大小和複雜度。

3

等待掃描完成

當進度條達到 100% 時，主動掃描就完成了。現在我們可以查看詳細的掃描結果。

主動掃描策略

ZAP 提供多種掃描策略：

  • 標準模式 - 平衡測試深度和速度
  • 快速掃描 - 快速檢查常見漏洞
  • 徹底掃描 - 深度測試所有參數
  • 攻擊模式 - 最積極的測試方式

常見漏洞類型

了解 ZAP 能夠檢測的各種漏洞類型對於有效使用工具至關重要。

SQL 注入

風險等級：嚴重

攻擊者可以透過輸入特殊字元操縱資料庫查詢，竊取或修改資料。

示例: ' OR '1'='1

跨站腳本 (XSS)

風險等級：高

攻擊者可以在網站上執行惡意腳本，竊取用戶資料。

示例: <script>alert('XSS')</script>

跨站請求偽造 (CSRF)

風險等級：高

攻擊者誘使用戶在不知情的情況下執行非預期操作。

安全設定錯誤

風險等級：中

不安全的伺服器設定、權限問題等。

敏感資訊洩漏

風險等級：中

錯誤訊息、註解或回應中暴露敏感資訊。

點擊劫持

風險等級：低

攻擊者使用透明層誘使用戶點擊惡意連結。

OWASP Top 10 對應

ZAP 能夠檢測 OWASP Top 10 中的大多數漏洞：

  • A01:2021 - 存取控制破壞
  • A02:2021 - 加密機制失效
  • A03:2021 - 注入攻擊
  • A05:2021 - 安全設定錯誤
  • A07:2021 - 識別與認證失效

解讀掃描結果

掃描完成後，最重要的部分是理解 ZAP 發現了什麼。讓我們來解讀「Alerts」標籤頁中的結果。

風險等級

ZAP 將發現的漏洞分為四個風險等級：

  • 高 - 需要立即關注的嚴重漏洞
  • 中 - 重要的安全問題，應該盡快修復
  • 低 - 潛在的安全問題，建議修復
  • 資訊 - 僅供參考的資訊，不一定是安全問題

漏洞詳情解讀

點擊任一警報，您可以查看詳細資訊：

1

描述

漏洞的詳細說明，包括攻擊原理和潛在影響。

2

解決方案

如何修復這個漏洞的具體建議。

3

參考資料

相關的技術文件和參考連結。

4

請求/回應

ZAP 用來發現漏洞的具體測試請求和網站回應。

生成報告

ZAP 可以生成多種格式的報告：

  • HTML 報告
  • XML 報告
  • JSON 報告
  • Markdown 報告

通過「Report」選單可以生成專業的安全評估報告。

進階使用技巧

身份驗證測試

ZAP 支援多種身份驗證方式：

  • 表單基礎驗證
  • HTTP 認證
  • Script-based 驗證
  • OAuth/OpenID Connect

API 掃描

ZAP 可以掃描 REST API 和 GraphQL 端點：

# 導入 OpenAPI 定義
檔案 → 導入 OpenAPI 定義

自動化掃描

使用 ZAP 的 API 進行自動化掃描：

# 基礎掃描指令
zap-baseline.py -t https://example.com

外掛程式擴展

ZAP 擁有豐富的外掛程式生態系統：

  • 被動掃描規則
  • 主動掃描規則
  • 匯出工具
  • 身份驗證助手

重要提醒：道德使用

請僅在您擁有或已獲得明確授權的網站上使用 ZAP 進行測試。未經授權的掃描可能是違法行為。這些技能應該用於保護系統安全，而不是攻擊他人。

白帽駭客守則：

  • 始終獲得適當的授權
  • 尊重隱私和資料保護
  • 負責任地披露發現的漏洞
  • 使用技能來改善安全性，而不是破壞它
